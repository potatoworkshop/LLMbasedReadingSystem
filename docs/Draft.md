# 基于大语言模型的动态阅读材料生成与难度调节系统（论文草稿大纲）

## 题目（暂定）

- 基于大语言模型的阅读篇章生成与难度调节系统设计与实现
- 面向分级阅读训练的动态阅读材料生成系统设计与实现

## 摘要（后写）

- 研究背景：教育场景需要大量、分级、可控的阅读材料，且材料应能随学习者水平动态调整。
- 问题：现有 LLM 方法常见“可读性控制不稳定、事实保真不足、输出格式不稳定、难度评价维度单一”等问题。
- 方法：设计并实现一个包含文章生成、指定难度调节、文章改写与统计分析的动态阅读材料系统。
- 结果（占位）：通过案例分析与后续量化实验，验证系统在可控性、可用性和工程完整性方面的改进。
- 贡献：提出基于多指标反馈的闭环难度调节方法，并提供可视化分析与数据归档机制。

## 1 绪论（Introduction）

### 1.1 研究背景与意义

在分级阅读、英语教学和阅读测评训练等场景中，教师与内容开发者需要持续提供不同主题、不同篇幅、不同难度层级的阅读材料。尤其在个性化学习与自适应训练场景中，同一主题往往需要多个难度版本，以便适配不同学习者的阅读水平和训练阶段。传统人工编写方式虽然质量可控，但通常存在成本高、周期长、更新速度慢等问题，难以满足高频内容供给与快速迭代的实际需求。

随着大语言模型（Large Language Model, LLM）在文本生成、改写与问答任务中的能力提升，利用 LLM 自动生成阅读材料成为一个具有现实可行性的方向。LLM 在主题扩展、文本改写和风格模仿方面表现出较强的灵活性，使其有潜力承担“阅读材料初稿生成”和“难度调整”的工作，从而降低人工编写负担，提高内容生产效率。

然而，在教育应用场景中，“能够生成文本”并不等同于“能够生成可用的阅读材料”。面向阅读训练的内容生成至少需要同时满足几个条件：其一，难度应具备可控性，能够适配不同阅读水平；其二，内容应尽可能保持事实一致，避免在改写或调节过程中出现信息偏移；其三，输出应具有可验证性，便于系统解析、审查和后续处理；其四，生成结果应可追踪、可归档、可比较，从而支持后续分析与迭代优化。

此外，当前英语阅读材料在实际使用中还存在更新缓慢、难度等级离散（如 A2/B1/B2）以及难以根据学习者能力实时适配的问题。基于这一背景，本文将 LLM 视为一种“动态阅读材料引擎（dynamic reading material engine）”，重点探索其在阅读训练场景中的可控生成与指定难度调节能力，并在系统层面补充归档与分析机制，以增强工程可用性与研究可评估性。

### 1.2 国内外研究现状（简述，后续扩展）

自动生成阅读材料与测评内容（Automated Item Generation, AIG）已有较长研究历史。传统方法通常依赖模板、规则或专家预先构建的内容模型，在可控性方面具有优势，但在主题扩展能力、语言表达多样性和跨场景迁移方面存在明显限制。随着神经生成模型与大语言模型的发展，研究重心逐步从规则驱动转向数据与模型驱动，使阅读篇章生成、文本改写和教育内容生成等任务获得更高的灵活性。

已有研究表明，LLM 可以在提示工程（prompt engineering）和 few-shot 示例的帮助下生成接近目标风格的阅读篇章，并在一定程度上实现可读性或难度控制。例如，相关工作已探索了基于目标读者描述、可读性标签或 Lexile 目标的生成与改写方案，说明 LLM 在教育文本生成方面具有较好的应用潜力。同时，部分研究也展示了 few-shot 提示对“目标难度对齐”和“内容保留”的改善作用。

尽管如此，现有工作仍存在一些共性问题。首先，基于提示词的难度控制往往只能实现“趋势性变化”，难以稳定命中预期难度区间。其次，LLM 输出格式稳定性不足，结构化解析失败会影响系统自动化流程。再次，在改写或分级生成过程中，内容保真与事实一致性仍是关键风险点。最后，许多研究主要依赖少数经典可读性指标进行评价，难以充分反映文本在信息密度、关键词分布和语义抽象程度方面的差异，且研究通常聚焦单一任务（仅生成或仅改写），缺少面向实际使用的系统化集成方案。

基于上述现状，本文的研究定位并非单纯比较不同模型的生成效果，而是面向动态阅读材料生成场景，设计并实现一个包含生成、改写、指定难度调节、数据归档与统计分析的完整系统，以支撑后续实验评估与工程落地。

### 1.3 研究问题（Research Questions）

围绕上述背景与研究现状，本文提出以下研究问题。第一，LLM 能否根据指定主题、目标篇幅和难度等级稳定生成可用的阅读文章（RQ1）。该问题关注系统的基础可用性，即生成结果是否能够在结构、长度和内容形态上满足后续处理与教学使用的基本要求。

第二，如何将“语言难度”转化为可计算、可比较的指标体系，用于生成评估与调节反馈（RQ2）。这一问题关注“难度”概念的操作化表达。如果难度仅停留在“简单/困难”的描述层面，就难以形成可重复、可验证的调节过程，也难以进行后续实验比较。

第三，如何将文章改写到指定难度等级（1–5），并使该等级由多项阅读指标的目标区间共同定义，而非仅依赖“简化/变难”的描述性指令（RQ3）。该问题是本文方法设计的核心，重点在于构建一种面向目标区间的闭环调节机制，使难度控制从“提示驱动”进一步走向“指标反馈驱动”。

第四，如何构建一个完整的阅读材料生成 pipeline，使生成、改写、难度调节、归档与分析能够连贯协同（RQ4）。该问题关注工程层面的系统性与复用性，旨在为后续实验评估、功能扩展与迭代优化提供统一平台。

### 1.4 本文主要工作与贡献（非量化版本）

针对上述研究问题，本文的主要工作与贡献如下。首先，设计并实现了一个面向动态阅读材料生成的 LLM 系统，覆盖文章生成、基础改写、指定难度调节、本地语料读取与统计可视化等功能，使研究工作不局限于离线实验脚本，而是形成可运行、可演示、可复用的工程平台。

其次，本文提出了一种基于多指标反馈的闭环难度调节流程。该流程将指定难度等级（1–5）映射为多项阅读指标的目标区间，在调节过程中通过迭代改写、指标重新计算与候选筛选实现逐步逼近目标难度，并结合内容保真度评分抑制事实偏移或信息丢失，从而在“难度命中”与“内容保持”之间取得平衡。

再次，本文在传统可读性指标基础上扩展了信息密度、关键词密度（TF-IDF）与抽象度等分析维度，用于更细致地刻画文本难度与信息结构差异。该设计有助于缓解仅依赖少数可读性公式进行评价的局限，为后续语料对比与结果分析提供更丰富的观察视角。

最后，本文构建了离线统计脚本与前端可视化展示页面，用于对多个数据集进行可读性、词汇密度与抽象度等维度的汇总与分布分析，为实验设计、结果展示和系统迭代提供统一的数据支持与观察界面。

### 1.5 论文结构安排

本文后续章节安排如下：第2章对相关工作进行梳理与比较，重点讨论 LLM 驱动的阅读材料生成、可读性控制与教育内容分级生成等研究，并在此基础上说明本文的研究定位与改进点。第3章介绍系统具体实现与关键模块，包括文章生成、基础改写、指定难度调节以及统计分析支持模块的实现要点。第4章给出本文的方法学与系统设计，明确应用场景、需求分析、总体架构与技术路线设计。

第5章给出实验设计与评估方案，说明实验对象、任务设置与评价维度，重点围绕生成稳定性、难度可计算化和指定难度调节能力展开。第6章展示实验结果并进行讨论，分析系统在不同任务与指标下的表现、优势与局限。第7章对全文工作进行总结，并给出后续改进方向与研究展望。

## 2 相关工作（Related Work）

### 2.1 LLM 驱动的阅读材料生成

自动生成阅读材料与测评内容的研究可以追溯到 AIG（Automated Item Generation）范式。早期方法通常依赖模板、规则和专家定义的内容结构，这类方法在输出格式和题型结构上具备较高可控性，但在主题扩展、多样表达和跨任务迁移方面受到明显限制。随着神经文本生成方法的发展，研究开始尝试以数据驱动的方式替代部分人工规则；而在大语言模型出现后，阅读材料生成进一步从“模板填充”转向“基于指令的开放文本生成”，在语言自然度和主题覆盖能力上获得明显提升。

在阅读篇章生成方面，`Automated reading passage generation with OpenAI’s large language model` 是与本文最相关的代表性工作之一。该研究利用 GPT-3 生成用于阅读测评的篇章，通过提示词设计引导输出接近目标风格，并结合目标读者年龄信息和文本难度筛选来选择候选结果，随后通过人工编辑与人工评审验证生成篇章的可用性。该工作的重要价值在于证明了 LLM 在阅读测评篇章生成上的实际可行性，也说明了提示工程和读者画像信息对生成质量具有显著影响。

然而，这类篇章生成工作仍存在若干局限。首先，其质量保障流程较依赖人工筛选、人工编辑和人工评审，系统自动化程度有限。其次，难度控制往往以外部筛选为主，而不是在生成与改写过程中形成可计算反馈闭环。再次，研究重点通常放在“生成篇章是否可用”的验证上，对于生成结果的持续归档、跨数据集统计比较和后续迭代分析支持相对不足。

此外，`Evaluating Reading Comprehension Exercises Generated by LLMs` 从系统应用角度展示了 LLM 在阅读理解练习生成中的潜力。该研究不仅关注阅读材料与题目本身的质量，还强调教师、学生和其他参与者的反馈，说明在教育场景中，“生成能力”需要与“系统设计”和“实际使用流程”共同考虑。对本文而言，这类工作提供的启示在于：阅读材料生成不应仅被视为单一模型任务，而应被放在一个完整教育系统或内容生产流程中进行设计和评价。

### 2.2 可读性控制与文本难度调节

文本难度调节相关研究与文本简化（text simplification）、可读性控制生成（readability-controlled generation）和教育分级文本生成（leveled-text generation）密切相关。传统文本简化方法较多依赖规则替换、词汇替换或监督学习模型，而 LLM 的出现使研究者能够直接通过提示词进行零样本或少样本的简化、复杂化和风格控制。相比传统方法，LLM 方案在跨领域适配和语言自然度方面具有优势，但在控制精度、内容保留和输出稳定性上也引入了新的挑战。

`Generating Educational Materials with Different Levels of Readability using LLMs` 提出了面向教育材料的 leveled-text generation 任务，研究目标是将已有文本改写到指定可读性等级（Lexile）并尽量保持原意。该研究比较了多个 LLM 在 zero-shot 与 few-shot 条件下的表现，并指出 few-shot 提示能够显著改善难度控制与内容保留效果。这一工作为教育分级文本生成提供了清晰的任务定义和评估思路，对本文具有直接参考价值。

同时，该研究也揭示了 LLM 分级改写中的典型问题，包括目标难度偏差、文本长度变化偏差、信息失真与编辑分布不均等。这些问题说明，仅依赖“把文本改写得更简单/更难”的描述性提示并不足以稳定实现指定难度目标，仍需要更明确的难度目标表达与过程约束。

`Free-text Rationale Generation under Readability Level Control` 虽然研究对象是模型解释文本（rationale），而非教育阅读材料，但其方法结论对本文具有重要启发意义。该研究通过提示词对不同可读性等级进行控制，并结合自动指标和人工评价分析结果，发现 LLM 输出在可读性上能够呈现趋势性变化，但提示等级与实际测得的可读性指标之间并不完全一致。该现象从侧面支持了本文的研究动机，即在教育文本场景中需要从“描述性等级控制”进一步走向“指标反馈驱动”的难度调节机制。

在更细粒度的文本简化评测层面，`BLESS: Benchmarking Large Language Models on Sentence Simplification` 对大量 LLM 的句子简化能力进行了系统基准测试，并从自动指标、编辑操作类型和人工分析三个角度进行评估。该研究表明，不同模型在简化质量和编辑策略上存在显著差异，提示研究者不能仅用单一分数评价“简化能力”。对本文而言，这意味着在设计阅读材料难度调节方案时，需要同时关注可读性变化、语言编辑行为和内容保留，而不是只观察某一个可读性公式。

### 2.3 教育内容生成与分级阅读应用

随着生成式 AI 在教育场景中的应用扩大，研究者开始从“模型是否能生成内容”转向“生成内容是否能服务教学目标”。在教育内容生成领域，LLM 已被用于阅读材料生成、练习题生成、个性化反馈和辅助教学系统设计。相关研究普遍指出，LLM 具有降低内容生产成本、提升个性化供给能力的潜力，但其在教学场景中的有效使用仍依赖合理的任务设计、人工监督与评价机制。

`Evaluating Reading Comprehension Exercises Generated by LLMs` 提供了一个较具代表性的系统化实践案例。该研究通过系统设计将 ChatGPT 集成到阅读理解练习生成流程中，并通过自动评价与人工评价（包括教育实践参与者反馈）分析其可用性。这类研究表明，教育场景中的 LLM 系统评估不仅包括文本质量，还包括系统流程设计、可用性和实际教学适配性。本文虽然不以阅读理解题目生成为主线，但借鉴其“系统化落地”和“多方评价”的研究视角。

在教育教学效果与课堂应用层面，`Investigating the Impact of ChatGPT on Reading Proficiency in EFL Context` 从 EFL 学习成效角度讨论 ChatGPT 对阅读能力提升的潜在作用，反映出生成式 AI 在真实教学环境中的应用价值与实践需求。`Research on Generative Artificial Intelligence in Senior High Schools’ English Reading Teaching` 则从高中英语阅读教学场景出发，强调 GAI 在丰富阅读资源、支持个性化教学和改进评价方式方面的潜力，同时指出技术依赖、教学理念转变和课堂互动质量等问题。这些研究共同说明：在分级阅读应用场景中，系统不仅要能生成文本，还需要考虑难度可控、内容可靠和评价机制支持。

此外，`Automated Educational Question Generation at Different Bloom’s Skill Levels using Large Language Models` 虽然聚焦教育问题生成，但其研究结论对教育内容系统设计具有旁证意义：一方面，LLM 在教育内容生成中可以通过提示策略产生较高质量输出；另一方面，教学质量评价仍明显依赖专家评估，自动评价与人工评价之间存在差距。该现象提示我们，在阅读材料生成系统中需要尽可能提升输出结构化、过程可追踪和质量可验证能力，以降低纯人工评审的负担。

综上，教育内容生成研究已经从“技术可行性验证”逐步走向“教学场景适配与系统化部署”，而分级阅读场景对难度控制、内容保留和评价支持提出了更高要求。本文的切入点是在这一背景下，将生成、改写、指定难度调节与统计分析能力集成为动态阅读材料系统，以服务后续实验评估和教学应用探索。

### 2.4 本文与现有工作的差异（本章结尾可作为过渡）

综合上述工作可以看出，现有研究已分别在阅读篇章生成、可读性控制、文本简化评测和教育应用系统化方面取得了重要进展，但仍较少有工作将这些能力整合为一个面向动态阅读材料场景的完整系统。相较于已有研究，本文的差异主要体现在以下几个方面。

第一，本文不仅关注阅读篇章生成本身，还引入了基于指标反馈的指定难度闭环调节机制。与仅通过提示词描述“更简单/更难”或仅在生成后筛选难度的方案不同，本文将难度等级（1–5）映射为多项阅读指标目标区间，并在改写过程中迭代计算指标与选择候选结果，使调节过程具备更强的可解释性与可操作性。

第二，本文在难度调节过程中加入内容保真约束，尝试在目标难度命中与信息保持之间取得平衡。现有分级生成与简化工作普遍强调内容保留的重要性，但在工程系统层面往往缺少明确的保真度筛选策略。本文通过对实体、数字和关键词等信息的覆盖情况进行评分，将保真度作为候选接受条件之一，以降低事实偏移和关键信息丢失风险。

第三，本文在评价与分析层面不仅使用经典可读性指标，还扩展了信息密度、关键词密度（TF-IDF）与抽象度等维度，用于更细致地刻画阅读材料的难度与信息结构差异。这一设计有助于弥补仅依赖少数可读性公式所带来的表征不足，为后续真实语料与生成语料对比提供更丰富的分析视角。

第四，本文不将研究工作限定为离线实验，而是实现了可交互系统、数据归档与统计可视化能力，使生成、改写、难度调节和分析能够在统一流程中运行。该系统化设计既服务于论文中的实验评估，也为后续功能扩展与实际应用探索提供基础。基于上述差异，下一章将先介绍系统实现与关键模块，随后在第4章进一步说明本文的方法学与系统设计思路。

## 3 系统具体实现（Implementation）

### 3.1 研究流程概述（Pipeline）

本研究的实现流程遵循“问题定义—文献调研—系统设计—模块实现—数据归档与分析—实验设计”的总体路径。首先，在问题定义阶段，本文将阅读篇章生成与难度调节抽象为“可控生成 + 指标反馈”任务，并据此明确系统需要同时支持生成、改写、难度调节和分析等能力。随后，通过对阅读篇章生成、可读性控制与教育内容生成相关工作的调研，提炼出本文的实现重点，即指定难度调节、内容保真约束与统计分析支持。

在此基础上，本文构建了前后端分离的动态阅读材料生成系统，并围绕文章生成、基础改写、指定难度调节、离线统计与可视化展示等模块完成实现。系统运行过程中会对生成与改写结果进行本地归档，以支撑后续统计分析、案例复查与实验复现。最后，基于系统输出数据与参考语料组织实验设计，用于验证生成稳定性、难度可计算化与难度调节效果。

### 3.2 总体方法流程（可写成流程图）

### 3.3 文章生成方法

#### 3.3.1 输入参数与约束

文章生成接口接收的核心参数包括主题（`topic`）、难度等级（`level`）、目标字数（`target_words`）、语言（当前实现为英文）以及可选模型（`model`）。这些参数共同决定生成任务的内容方向、篇幅规模和目标难度定位。其中，主题用于约束语义范围，目标字数用于控制篇章长度，难度等级则用于在提示词中加入难度描述和目标提示信息。

为保证系统稳定运行，后端在进入生成逻辑前对输入参数进行边界约束和类型校验。例如，等级被限制在 1–5 的整数范围内，目标字数需在预设区间内，语言字段需符合当前支持的枚举值。该设计能够在请求入口阶段拦截明显无效输入，减少后续模型调用与解析失败带来的资源浪费。

#### 3.3.2 提示词设计

文章生成提示词采用约束式设计思路，在同一提示中同时给出主题要求、目标字数及可接受范围、难度等级描述、文体要求和输出格式约束。具体而言，系统会根据用户输入的目标字数计算一个可接受长度范围，并在提示词中明确给出“目标长度”和“优选长度区间”；同时结合难度等级配置提供相应的难度标签与语言风格提示，以引导模型生成更符合预期的文本。

在输出形式上，系统要求模型返回严格 JSON（包含标题和正文），并明确禁止额外的 Markdown、列表或无关文本。该设计并不能完全消除模型输出异常，但能够显著提高结构化解析成功率，并为后续指标计算、归档和前端展示提供统一输入格式。

#### 3.3.3 候选选择与归档机制

生成结果返回后，系统会先对正文文本计算基础指标，包括词数、句数、平均句长及若干可读性指标。这些指标一方面用于前端即时展示，另一方面作为候选筛选和后续难度调节的基础数据。考虑到 LLM 在长度控制上的波动性，系统支持多次生成尝试，并在尝试过程中保留“最接近目标字数”的候选结果，以提升最终输出的稳定性和可用性。

在候选确定后，系统会将文章标题、分段正文、生成参数、指标、模型信息与生成时间戳等内容写入本地归档文件。归档机制不仅服务于结果保存，也为后续统计分析、案例对照和实验复现提供数据基础，是系统从“单次生成工具”转向“研究与工程平台”的关键实现环节。

### 3.4 文本改写与难度调节方法

#### 3.4.1 基础改写模式

系统实现了三类基础改写模式：简化（`simplify`）、复杂化（`harder`）与缩短句子（`shorten`）。其中，简化与复杂化主要用于观察词汇层级变化和语言表达复杂度变化，缩短句子模式则侧重通过拆分长句与简化从句结构来降低句法负担。这些模式既可作为用户交互界面中的快速操作，也可作为后续指定难度调节的直观对照基线。

在实现层面，基础改写通过独立服务模块构造模式化提示词，并要求模型返回统一的 JSON 结构。相较于指定难度调节，基础改写不依赖目标指标区间与多轮筛选，因此流程更轻量，适合用于交互展示和初步实验观察。

#### 3.4.2 难度等级配置（Difficulty Profile）

指定难度调节的核心前提是对“难度等级”进行可计算表达。为此，系统将难度离散为 1–5 级，并为每一级定义一组阅读指标目标区间，至少包括 Flesch-Kincaid Grade、Flesch Reading Ease、平均句长和复杂词比例。该配置存放于独立的难度等级配置模块中，便于后续统一调整和实验比较。

与使用单一指标或单点目标值相比，采用“多指标 + 目标区间”的方式更符合 LLM 文本生成与改写的实际波动特性。一方面，多指标能够从不同角度描述文本难度；另一方面，目标区间允许一定范围内的偏差，提高系统在实际运行中的鲁棒性与可操作性。

#### 3.4.3 闭环难度调节策略

指定难度调节采用闭环迭代策略。系统首先计算输入文本的当前指标，并根据其与目标等级区间的偏差判断调节方向（趋向更易或更难）。随后，系统调用 LLM 执行一轮改写，并重新计算候选文本的指标表现，以评估其是否进入目标区间，或是否相较当前版本更接近目标区间。

若候选文本达到目标区间，系统可提前结束迭代并返回结果；若尚未命中目标但相对更优，则接受该候选作为下一轮迭代的基础文本。该策略使难度调节过程具备明确反馈路径，而不是单次调用模型后直接接受结果。当达到最大轮次仍未完全命中目标时，系统输出当前最佳候选（best effort），以保证流程可终止且结果可用。

#### 3.4.4 内容保真约束（关键改进点）

难度调节过程中，LLM 可能通过改写引入事实偏移、删除关键信息或替换重要表达，从而导致文本虽然在可读性指标上更接近目标，但在语义内容上偏离原文。为降低这一风险，系统在调节流程中引入内容保真约束，构建基于实体、数字与关键词覆盖情况的保真度评分机制。实体召回用于约束命名实体信息的保留，数字召回用于保护日期、数量、百分比等关键事实，关键词召回则用于补充对主要内容词的覆盖检查。

在候选筛选时，系统不仅比较难度目标的接近程度，还要求候选文本的保真度达到设定阈值。只有当候选文本在保真度合格的前提下实现目标命中或距离改善时，才会被接受。该机制形成了“难度—保真”折中策略，使系统在追求目标等级对齐的同时，尽量保持原文信息结构与核心事实。

### 3.5 数据归档与统计分析方法

#### 3.5.1 数据归档

系统将生成文章与改写/难度调节结果分别写入不同本地目录，以便区分数据来源和处理阶段。归档内容通常包括原始文本或输入文本、结果文本、分段结构、指标信息、模型信息、参数配置与时间戳等元数据。对于难度调节任务，系统还会记录调节过程相关信息（如轮次和最终命中情况），便于后续案例分析。

这种归档方式一方面支持前端本地语料浏览功能，另一方面也为离线统计脚本和实验分析提供稳定输入。相比仅在内存中展示结果的原型系统，归档机制显著提高了系统的研究复现性和工程可维护性。

#### 3.5.2 统计指标设计（非量化章节先描述思路）

为支持后续实验中的语料对比与结果分析，系统设计了离线统计指标体系。除常见的可读性指标（如 FRE、FK、ARI、Coleman-Liau、Gunning Fog）外，本文还引入信息密度指标（如 TTR、Root TTR）、关键词密度（基于 TF-IDF 的关键词统计）以及抽象度指标（基于词汇具体性词典估计）等维度，用于更细致地刻画文本的难度与信息结构。

这些指标并不直接替代人工评价，而是为系统提供可计算、可比较的分析基础。尤其在比较真实语料、生成语料与难度调节语料时，多维指标设计能够帮助研究者从不同角度观察文本变化趋势，减少对单一可读性公式的过度依赖。

#### 3.5.3 可视化分析支持

在统计实现基础上，系统提供了面向多数据集的可视化分析支持，包括汇总表、分布图和明细表格等展示形式。可视化模块能够按数据集整体统计，也可按 passage 分组查看分布差异，便于分析参考语料与系统生成语料在不同指标上的变化趋势。

该设计使统计结果不再局限于脚本输出文件，而能够直接服务于论文中的图表整理与结果讨论，同时提升系统在演示和迭代开发阶段的可观察性。

### 3.6 系统界面与交互流程说明

系统前端提供统一的交互入口，支持从参数输入到结果查看的完整流程。首页界面支持主题、目标字数、等级与模型选择等参数配置，并可触发文章生成、基础改写和指定难度调节任务。对于难度调节结果，界面会展示调节前后的关键指标变化以及保真度信息，以帮助用户快速判断调节效果。

此外，系统还提供本地数据浏览与统计分析页面。本地数据浏览界面可加载生成语料与参考语料进行对照查看，便于进行案例分析；统计分析页面则展示数据集汇总、分布图和分组对比结果，为实验设计和结果汇报提供直观支持。通过这些界面设计，系统实现了“生成—查看—比较—分析”的基本交互闭环。

## 4 方法学与系统设计（Methodology and System Design）

本章从方法学与系统设计两个层面说明本文的整体方案。一方面，本文将阅读材料生成与难度调节视为“可控生成 + 指标反馈”的方法问题，需要明确目标难度如何表示、如何在生成与改写过程中进行反馈与筛选；另一方面，本文同时面向可运行系统进行设计，因此还需要从应用场景、功能需求和系统架构角度组织各模块，以支撑后续实现与实验评估。

### 4.1 应用场景与用户需求

本文系统面向分级阅读训练与教学资源辅助生成场景，主要用户可包括教师、教学研究者以及需要构建阅读训练语料的开发者。在实际使用中，用户通常需要围绕某一主题快速生成可读的英文阅读文章，并根据不同学习者水平调整文章难度与篇幅。相比一次性生成固定文本，这类场景更强调内容的动态调整能力与持续供给能力。

结合前文的研究背景与文献分析，本文将用户需求概括为三个核心场景。第一类场景是按主题、字数和目标难度快速生成阅读文章，用于教学补充材料、训练样本或实验语料构建。第二类场景是对既有文章进行改写与难度调节，包括基础的简化、复杂化和句子缩短，以及面向指定等级（1–5）的目标难度调节。第三类场景是对生成结果进行本地归档、查看和统计分析，以支持后续的语料对比、实验复现和系统迭代。

从方法学角度看，这些场景共同指向一个核心需求：系统不仅要生成文本，还要支持对“难度”进行可计算表达和可重复控制。因此，本文在需求设计阶段就将“指定难度调节”和“统计分析支持”视为核心能力，而不是附加功能。

### 4.2 功能需求

基于上述应用场景，本文系统的功能需求主要包括文章生成、文本改写、指定难度调节、本地语料读取与展示，以及数据统计与可视化分析。文章生成功能需要支持主题、目标字数、难度等级和模型选择等参数输入，并返回结构化结果，以便后续自动处理。文本改写功能需要支持 `simplify`、`harder` 和 `shorten` 等基础模式，用于快速观察改写效果和构造对照样本。

指定难度调节功能是本文的核心功能之一。与基础改写不同，该功能需要根据目标等级（1–5）对应的阅读指标区间进行多轮迭代调节，并在调节过程中评估文本是否更接近目标区间，同时兼顾内容保真。该功能直接对应本文的方法学目标，即将“难度控制”从描述性指令转化为可计算、可反馈的系统流程。

此外，考虑到论文实验与后续扩展需求，系统还需要提供本地语料读取与展示能力，以及多数据集统计分析与可视化能力。前者用于快速查看生成结果与参考语料，后者用于对不同来源语料进行可读性与词汇特征对比，为实验设计和结果分析提供基础数据支持。

### 4.3 非功能需求

除功能需求外，本文系统还需满足若干非功能需求，以保证其可维护性、可扩展性和可复用性。首先，在可维护性方面，系统采用前后端分离与模块化服务设计，将文章生成、难度调节、LLM 适配、指标计算和统计分析等能力拆分为相对独立的模块，便于后续修改与功能扩展。

其次，在可扩展性方面，系统需要支持不同的 LLM 提供方与模型配置。为此，本文通过统一适配层封装模型调用逻辑，使上层业务模块无需直接依赖某一特定 API，从而支持 OpenRouter 与 Ollama 等不同运行方式。这一设计有助于后续进行模型替换、对比实验或本地部署。

再次，在可追踪性与研究可评估性方面，系统要求对生成与改写结果进行归档，并记录指标、模型信息和时间戳等元数据，以便开展后续统计分析、错误排查与实验复现。最后，在可用性与基础安全性方面，系统前端需支持参数配置、结果展示和统计查看等基本交互流程，本地文件读取接口需具备基础的文件名校验能力，以降低路径穿越等常见风险。

### 4.4 系统总体架构（图占位）

基于上述需求，本文设计了一个由前端交互层、后端服务层、LLM 适配层、数据层与统计分析层组成的系统总体架构。前端部分采用 Next.js 实现，负责参数输入、结果展示、本地语料浏览和统计页面呈现；后端部分采用 Express + TypeScript 实现，负责文章生成、改写、难度调节等业务逻辑与接口编排。

在模型调用方面，系统通过 LLM 适配层对 OpenRouter 和 Ollama 进行统一封装，使文章生成与难度调节等服务模块能够通过一致的接口调用不同模型。数据层用于保存生成文章、改写结果及参考语料（如 `out_generated`、`out_simplified`、`out_questions`），为系统展示和实验分析提供基础数据来源。统计分析层则由离线统计脚本与前端统计页面共同构成，负责计算并展示可读性、信息密度、关键词密度与抽象度等指标。

从方法学视角看，该架构的关键价值在于将“生成—调节—归档—分析”串联为统一流程：前端触发任务，后端完成生成与调节，结果写入数据层，再由统计分析层进行汇总与可视化。这种设计既支持论文中的实验评估，也为后续系统扩展与教学场景应用提供基础。

## 5 实验设计（仅写设计，不写结果）

### 5.1 实验目标

- 验证系统是否能在工程上稳定完成“生成-改写-指定难度调节-归档-分析”的完整流程。
- 评估难度调节模块在“目标命中”和“内容保真”之间的平衡表现。
- 对比系统生成数据与参考语料在可读性与词汇特征上的分布差异。

### 5.2 实验对象与数据来源

- 本地参考语料：`out_questions/`（已有阅读文章数据）
- 系统生成语料：`out_generated/`
- 难度调节结果语料：`out_simplified/`
- 补充案例数据：按不同主题、等级、目标字数生成的新样本（后续补充）

### 5.3 实验任务设计

#### 5.3.1 任务A：阅读文章生成可用性验证

- 输入不同主题、等级、目标字数
- 检查生成成功率、结构解析成功情况、归档是否完整
- 记录生成结果是否满足基本长度与格式要求

#### 5.3.2 任务B：真实阅读材料与生成材料的可读性对比实验

- 以 `out_questions/` 作为参考阅读材料语料，以 `out_generated/` 作为系统生成语料
- 对比经典可读性指标与扩展词汇/语义指标的分布差异
- 分析系统生成语料与参考语料在难度区间上的对应关系与偏差

#### 5.3.3 任务C：难度调节案例实验

- 选取若干生成文章或参考文章作为输入
- 设定不同目标等级（升难/降难）
- 观察迭代轮次、是否命中目标等级、保真度变化
- 记录 best effort 情况，用于分析失败或部分成功原因

#### 5.3.4 任务D：多数据集统计对比实验（扩展）

- 使用统计脚本生成 `article_stats.json`
- 对比 `out_questions`、`out_generated`、`out_simplified` 的分布差异
- 结合可视化页面进行整体与分组分析

### 5.4 评价维度与指标设计（先写维度，数值后填）

#### 5.4.1 文章生成维度

- 格式有效性（是否可解析为目标 JSON）
- 长度控制表现（是否接近目标字数范围）
- 文本可读性与难度特征（多指标）
- 人工主观评价（可选，小样本：通顺性、连贯性、适配性）

#### 5.4.2 难度调节维度

- 目标难度命中情况（命中/未命中）
- 难度接近程度（距离目标区间的偏差）
- 内容保真度（实体/数字/关键词召回）
- 迭代效率（轮次）

### 5.5 对比方案（与文献的关系）

- 与文献对比的重点放在“方法机制与系统能力”而非仅数值高低：
  - 是否支持闭环难度调节
  - 是否具备保真约束
  - 是否将指定难度映射为可计算指标区间
  - 是否支持可视化统计分析与数据归档
- 对于数值对比（如可读性控制效果），仅在相同或相近任务设定下进行谨慎比较，避免不公平结论。
